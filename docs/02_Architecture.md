# Technical Architecture (Pipeline & Stack)

## Core Philosophy
This tool requires **zero direct integration** with private practice software (EMIS, SystmOne). It relies exclusively on public data released by NHS England Digital, processed through a cheap, serverless ETL (Extract, Transform, Load) pipeline.

## 1. Extract, Transform, Load (The Back-End)
Instead of renting a server or database, we use **GitHub Actions**.
* **Extract:** A Python script triggered by a GitHub Action cron schedule (e.g., monthly) scrapes public CSVs from NHS Digital:
    * Appointments in General Practice (for DNA rates).
    * Patients Registered at a GP Practice (for List Size and Demographics).
    * Quality and Outcomes Framework (QOF) Data (for performance indicators).
* **Transform (Data Science Layer):** The Python script cleans the data and runs a K-Means clustering algorithm. This algorithm groups practices together into "Archetypes" based on list size and local demographics.
* **Load:** The script compiles the final, crunched numbers into flat `data.json` files and pushes them as a commit directly into the GitHub repository.

## 2. The Front-End (The Application)
The application simply reads the static `.json` files generated by the GitHub Action. This makes the tool lightning-fast and free to host.
* **Framework:** Next.js or React.
* **Styling:** Tailwind CSS (Focus on high-contrast, minimalist, elegant design).
* **Interactivity & Storytelling:** D3.js, GSAP (ScrollTrigger), or Scrollama.js. The user should "scroll" through their personalized report, triggering elegant chart animations and text reveals to reduce cognitive load. 
* **Hosting:** Vercel, Netlify, or GitHub Pages.
